# Deskflow-Plus 设计背景

## 项目定位

Deskflow-Plus 是一个参考 Deskflow 核心原理重新设计的新项目，目标是在局域网场景下提供低延迟、稳定、安全的多机共享一套键盘和鼠标的体验，并在此基础上逐步增加图片剪贴板、文件传输、鼠标灵敏度调节、滚轮平滑调节等增强能力。

项目首发平台仅支持 Windows 和 macOS，不以 Linux 兼容为目标。该取舍用于集中资源优化双平台输入链路、权限模型、打包交付和长期维护成本。

## 参考来源

Deskflow-Plus 的产品交互和部分系统能力设计明确参考了 Deskflow。

参考范围包括：

- 多机共享一套键盘和鼠标
- 通过屏幕边界切换控制目标设备
- 配对后通过拖拽配置主副屏幕的相对位置
- 剪贴板共享
- 局域网内建立连接并维持稳定会话

参考原则：

- 参考核心原理和交互模型
- 不复制原项目实现
- 不要求二进制或源码兼容

## 为什么不直接沿用当前 Deskflow 仓库

虽然当前 Deskflow 仓库已经具备跨平台输入共享、剪贴板共享、平台层实现和打包脚本，但本项目决定采用新的技术栈重新设计，原因如下：

- 需要一个更适合长期二次开发的技术架构
- 需要聚焦 Windows 和 macOS，而不是继续承担更广的平台兼容负担
- 需要在安全配对、文件传输、滚轮平滑和灵敏度调节方面建立新的统一设计
- 需要降低核心模块的内存安全风险和后续维护复杂度
- 需要将高频输入逻辑、网络会话、安全模型和 UI 彻底按新边界重构

## 产品目标

Deskflow-Plus 的长期目标是：

- 保持 Deskflow 的核心价值，即多机共享一套键盘和鼠标
- 保持拖拽式屏幕拓扑配置体验
- 强化局域网环境下的安全配对和设备信任
- 增加图片剪贴板共享
- 增加应用内文件发送能力
- 增加多机鼠标灵敏度调节
- 增加多机滚轮平滑调节

## 设计原则

- 输入链低延迟优先
- 安全默认开启，不信任局域网
- 文件传输不得阻塞键鼠输入
- 平台差异下保持用户体验一致
- 首发版本只做可发布 MVP，不做过量扩展
- 所有关键行为都需要有测试与可追踪任务状态

## 发布策略

Deskflow-Plus 按两阶段推进：

### MVP 首发范围

MVP 只包含可发布的一版核心能力：

- Windows 和 macOS 双平台
- 单主控、多客户端模式
- GUI 配置界面 + 后台常驻核心进程
- 设备自动发现与手工添加
- TLS 配对与设备信任
- 固定网格拖拽式屏幕拓扑
- 键盘鼠标共享
- 文本剪贴板共享

### Post-MVP 增强范围

以下能力保留在首个可发布版本之后推进：

- 图片剪贴板共享
- 应用内文件发送
- 鼠标灵敏度调节
- 滚轮平滑调节

## MVP 明确不包含

- Linux
- Windows UAC/登录屏幕等安全桌面控制
- macOS 登录前会话或系统级特权助手能力
- 系统级拖拽传文件到远端桌面
- 共享目录同步
- 外网穿透或云中继
- 富文本全格式透传
- 任意像素级异形屏幕拼接
